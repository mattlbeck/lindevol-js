fix: cap telemetry data to prevent OOM crash on long runs

After running for extended periods, the SimData arrays grew without bound,
eventually causing `Array buffer allocation failed` in the worker when
allocating the next pixel buffer frame.

Added a rolling window cap of 2000 data points per series in `SimData.recordStep()`.
When the limit is exceeded the oldest entry is shifted off, keeping memory usage
stable across arbitrarily long simulation runs. Charts now display a sliding
window of recent history rather than the entire run.
