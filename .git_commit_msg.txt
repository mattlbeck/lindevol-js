feat(ui): improve cell info display with 9-cell pictograms and rule matching

- Rewrote `sendCellInfo` in `simulation.worker.js` to correctly determine the active matching rule depending on the active genome interpreter (Block vs Promotor).
- Modified the worker to send structured rule data (including equality masks and action directions) rather than just formatted strings.
- Completely redesigned `renderCellInfo` in `src/index.js` to render the cell's current state and each genome rule as a 9-cell grid pictogram using HTML/CSS.
- Visuals:
  - Green squares denote required filled space.
  - Subtly patterned squares denote "don't care" ignored spaces.
  - Bright yellow glowing squares indicate the target direction of a divide action.
- Added a collapsible "Pictogram Legend" info box to explain the display.
- Matched genome rules are now highlighted with a green background to indicate the active rule.
- Added a 10px margin to the rule action labels to keep them close to the index rather than right-aligned.
