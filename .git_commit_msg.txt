fix: rate-limit telemetry updates to prevent UI freezing

- Abstracted the `self.postMessage` telemetry dispatch into a `pushStats()` function inside `simulation.worker.js`.
- Moved `pushStats()` out of the high-frequency `doStep()` loop and synced it with the `pushFrame()` timing logic (capped at ~60 FPS).
- The worker still records data points at the user's defined `record_interval`, but buffers the data to prevent flooding the main thread's message queue.
- This ensures the UI (specifically the "Stop" button) remains fully responsive even when running thousands of steps per frame.
